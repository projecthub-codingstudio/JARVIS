# ProjectHub DESIGN.md

## Product Character
ProjectHub is a macOS-native AI workspace for developers.
It should feel serious, quiet, efficient, dense, and trustworthy.

## Core Metaphor
- Not a chatbot app
- Not a marketing dashboard
- Not a generic admin console
- It is a developer operating surface

## Product Pillars
- Command-first workflow
- Repository-aware navigation
- Evidence-based answers
- Fast document reading and verification
- AI embedded into the work surface, not visually dominant

## Layout Rules
- Prefer split panes over stacked cards
- Keep navigation narrow and stable
- Make the central work surface dominant
- Keep citations, metadata, and related files in a right-side inspector
- Use toolbars and panels, not hero sections
- Avoid excessive empty space

## Navigation Model
- Terminal
- Repository
- Documents
- Admin

## Information Hierarchy
- Primary: current task surface
- Secondary: repository or result list
- Tertiary: evidence, metadata, and related context

## Visual Style
- Desktop software, not web dashboard
- Calm, technical, editorial
- Minimal decorative chrome
- Strong alignment and spacing rhythm
- Subtle borders, restrained shadows
- Very limited glow

## Color Direction
- Graphite and warm gray base
- Off-white or soft neutral content surfaces
- Deep green accent
- Amber warning state
- Steel blue metadata or secondary emphasis
- Avoid purple-heavy AI styling

## Typography
- Crisp, modern, compact
- Strong hierarchy through size and weight
- Monospace for paths, metadata, citations, and terminal elements
- Avoid oversized headings

## Interaction Principles
- Keyboard-first
- Fast transitions
- Low friction between search, inspect, and open
- Distinguish clearly between repository browsing and result-document viewing

## Repository Screen
- File and path oriented
- Tree, grouped paths, or explorer patterns are preferred
- It should feel like a place to inspect source locations

## Documents Screen
- Result-document oriented
- Fast open, quick preview, and deep reading
- Breadcrumbs and metadata should reinforce that the user is viewing a selected result

## Terminal Screen
- Input should be prominent but compact
- Output should feel structured, not like a chat bubble feed
- Blocks, sections, and evidence links should be easy to scan

## Avoid
- Generic SaaS card grids
- Oversized KPI blocks
- Glassmorphism-heavy treatment
- Futuristic neon cyberpunk styling
- AI assistant as the visual center of the app

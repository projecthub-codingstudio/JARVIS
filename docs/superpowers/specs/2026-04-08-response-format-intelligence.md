> 작성일: 2026-04-08

## 1. 문제 분석

### 현재 상태
- 시스템 프롬프트: "핵심 답변만 1~3문장으로 간결하게 답하세요"
- **모든 질문에 서술형** 응답 강제
- 포맷 선택 가이드 없음

### 사용자 불만 시나리오

| 질문 유형 | 기대 포맷 | 현재 응답 |
|----------|----------|----------|
| "3일차 저녁 메뉴는?" | `닭가슴살 샐러드` (단답) | 3일차 저녁 메뉴에 대해 설명드리겠습니다... |
| "연결 상태 종류 알려줘" | 번호 목록 | 연결 상태에는 disconnected, connecting... (서술형) |
| "struct와 class 차이" | 비교 표 | struct와 class의 차이점은 값 타입과 참조 타입... |
| "빌드 방법" | Step 1, 2, 3 | 빌드를 하려면 먼저... |

## 2. 해결 전략

### 2.1 시스템 프롬프트 기반 포맷 자동 선택 (구현 완료)

LLM에게 질문 유형별 응답 포맷 규칙을 명시적으로 제공:

```
응답 포맷 규칙 (질문 유형에 따라 자동 선택):
- 구체적 값 → 값만 바로
- 목록 → 번호/불릿 목록
- 비교/차이 → markdown 표
- 방법/절차 → 단계별
- 설명/분석 → 구조화된 서술형
- 코드 → 코드 블록
- 예/아니오 → 한 문장
```

**장점**: 코드 변경 최소 (시스템 프롬프트만), LLM이 유연하게 판단
**한계**: LLM 모델 능력에 의존, 소형 모델은 지시 준수율이 낮을 수 있음

### 2.2 쿼리 의도 기반 포맷 힌트 주입 (향후)

Planner가 질문 유형을 분류하고, 포맷 힌트를 시스템 프롬프트에 동적 주입:

```python
# planner.py에서 format_hint 분류
if query has 비교 키워드 → format_hint = "table"
if query has 목록 키워드 → format_hint = "list"
if query has 방법 키워드 → format_hint = "steps"
...

# system_prompt에 동적 추가
f"이 질문에는 {format_hint} 형태로 답변하세요."
```

**장점**: LLM 부담 감소, 포맷 정확도 향상
**구현 시기**: 시스템 프롬프트 방식의 효과 관찰 후

### 2.3 응답 후처리 포맷 변환 (향후)

LLM 응답을 후처리하여 포맷 강화:

```python
# 서술형으로 나열된 항목 → 자동 목록 변환
# "A는 B이고, C는 D이며, E는 F입니다" → 불릿 목록
# 반복 구조 감지 → 표 변환
```

**장점**: 모델 독립적
**한계**: 복잡한 파싱 로직, 오변환 위험

## 3. 질문 유형 분류 체계

| 유형 | 키워드 패턴 | 기대 포맷 |
|------|-----------|----------|
| 단답형 | "뭐야", "알려줘", "뭔가요" + 구체적 대상 | 값 + 1문장 |
| 목록형 | "종류", "항목", "목록", "리스트", "어떤 것들" | 번호/불릿 목록 |
| 비교형 | "차이", "비교", "vs", "다른 점" | markdown 표 |
| 절차형 | "방법", "어떻게", "하는 법", "순서", "단계" | Step 1, 2, 3 |
| 분석형 | "설명", "분석", "구조", "아키텍처" | 구조화된 서술 |
| 코드형 | "코드", "함수", "구현", "예제" | 코드 블록 포함 |
| 확인형 | "맞나요", "가능한가요", "있나요" | 예/아니오 + 근거 |

## 4. "1~3문장" 제한 제거의 영향

### 이전
```
"핵심 답변만 1~3문장으로 간결하게 답하세요"
→ 모든 응답이 강제로 짧음
→ 상세 분석 요청도 3문장으로 잘림
→ 표/목록 형태 출력 불가
```

### 이후
```
질문 유형별 자동 포맷 선택
→ 단답: 여전히 간결
→ 목록/표: 구조화된 출력
→ 분석: 충분히 상세
→ 바로 본론 (서론 제거)
```

### 모니터링 필요
- 응답이 너무 길어지지 않는지 (특히 음성 응답)
- 포맷 지시 준수율 (EXAONE vs Gemma)
- 사용자 피드백 (👍/👎) 변화 추이
